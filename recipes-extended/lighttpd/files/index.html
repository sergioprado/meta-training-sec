About the Embedded Security training

A practical, deep-dive course for engineers building secure embedded Linux products.

The training starts by revisiting core security concepts like confidentiality, integrity and authenticity, and then moves into threat modeling tailored to embedded systems. You will analyze real devices, attacker profiles and typical vulnerabilities found in the field.

From there, the agenda covers secure coding, exploit techniques and modern mitigation strategies, cryptographic fundamentals, secure data-at-rest and data-in-transit, secure boot and chain of trust, access control mechanisms, sandboxing and isolation techniques, Trusted Execution Environments such as OP-TEE, network security, secure OTA updates and hardware security considerations.

Labs are performed directly on boards or in QEMU, using the same tools and workflows you would use in a real project: Yocto Project, Linux kernel, bootloaders, firewalls, VPNs, OP-TEE and more. The goal is to build an end-to-end view of how to design, implement and maintain secure embedded systems.

For more information about target audience, prerequisites, training format and registration options, please refer to the official training page on the Embedded Labworks website.
